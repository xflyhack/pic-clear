# pipe_gui.exe 使用说明

`pipe_gui.exe` 是 `pipeline.exe` 的**图形前端**，给不习惯敲命令行的同事用。
双击即可打开，表单选一下就能跑，也支持**最小化到系统托盘**、**全局快捷键呼出状态浮层**。

底层还是复用 `pipeline` 的编排逻辑，跑出来的任务、日志、状态**完全兼容** `pipeline.exe status/list/logs/stop`。

## 目录布局

```
C:\Windows\System32\
├── extract_frames.exe   （不变）
├── dedupe_pic.exe       （不变）
├── pipeline.exe         （可选，命令行入口）
├── pipe_gui.exe         （新增，双击运行，约 15-25 MB）
└── license.lic          （必须，跟其他 exe 共用）
```

`pipe_gui.exe` **可以放在任何目录**双击运行，它启动时会依次找：
1. `C:\Windows\System32\`
2. `pipe_gui.exe` 同目录
3. `PATH`

只要 `extract_frames.exe` / `dedupe_pic.exe` / `license.lic` 三个文件里**有一个位置齐全**，就能用。

## 使用步骤

1. **双击 `pipe_gui.exe`**，主窗口打开
2. **环境检测**：确认三个必要文件都是 `[√]`
3. **选数据盘**：默认自动挑 `Z:`，也可以下拉切换 `C:/D:/E:/Y:` 等
4. **选源目录**：如果盘上只有一个 `sjbz_*`，会自动填；否则点『浏览...』手工选
5. **选输出根**：默认 `<数据盘>\切帧结果`，可改
5b. **选 Marker 根**：默认 `<数据盘>\pic-clear-markers`，多机共享盘时**所有机器**
    都应指向同一位置（一般就是共享盘上某个固定目录）
6. **勾选子目录**：点『扫描/刷新』后出现复选框，默认全选，也支持『全选/全不选』
7. **调阈值**：
   - 相似度 `-t`（默认 3，越大删得越多）
   - 车运动 `-m`（默认 0.12，越小车稍微动就保护）
   - 抽帧 `--fps`（默认 1.0）
   - `抽帧并发数`（默认 1）：一台机器同时抽多少个视频，推荐 4-8。
     多机共享盘并发也安全（`_extract.lock` 互斥）
   - `抽帧锁 TTL(s)`（默认 900）：多机场景下崩溃 / 断网锁被视为过期的时间
   - `去重并发数`（默认 1）：watcher 同时跑多少个 `dedupe_pic.exe`。dedupe 内部
     YOLO 会用多核，2-3 通常够；太大反而互抢 CPU
   - `去重锁 TTL(s)`（默认 900）：同上，作用于 `_dedup.lock`
8. **勾选删除策略**：
   - `真删除` = 直接删（不勾则 dry-run 只出报告）
   - `永久删除` = 不落 `_trash` 目录
9. **调高级参数**（一般不用动，默认已经合理）：
    - `场景保护 -S`（默认开）：纯色屏/渐变屏这种传感器遮挡异常帧强制保留
    - `当日剩余上限 -L`（默认 80000）：累计剩余到这个数 pipeline 自动停，防止把有用图删过头
    - `watcher 扫描秒`（默认 3.0）：去重 watcher 多久扫一次 `_done.marker`
10. **注册快捷键**（可选，默认 `ctrl+alt+p`），点『注册快捷键』
11. **点『运行』** —— 任务后台起，主窗口不受影响

> 上面所有参数（包括源目录 / 输出根 / 勾选过的子目录）**下次启动会自动填回来**，
> 不用每次重新选。想清零就点『重置配置』。

## 主窗看进度（不用点『查看状态』）

任务跑起来后，主窗口会**实时刷进度**，不需要打开状态浮层：

- 子目录框顶部**汇总条**：
  ```
  当前任务: job_20260714_152012  ·  running  ·  完成 3/12  失败 0  ·  抽帧 47  去重 42  ·  抽帧子目录 [4/12] sub_004
  ```
- 子目录列表每一行**右侧进度**（跟着颜色变）：
  ```
  [√] sub_001            抽帧完   抽帧=8   去重=8    ← 绿色
  [√] sub_002            抽帧完   抽帧=6   去重=6    ← 绿色
  [√] sub_003            抽帧完   抽帧=7   去重=7    ← 绿色
  [√] sub_004            抽帧中   抽帧=3   去重=2    ← 橙色（跑着的这一个）
  [√] sub_005            待处理   抽帧=0   去重=0    ← 灰色
  [√] sub_006            失败     抽帧=0   去重=0    ← 红色
  ```
- 5 秒自动刷一次，跟主按钮『▶ 运行 / ■ 停止当前任务』的切换是同一个定时器

## 后台运行

- 勾选『点 × 时最小化到托盘（不退出）』（默认勾选）后：
  - 关闭主窗口 = 缩到系统托盘，不退出
  - 想真退出用『托盘右键 → 退出』或主界面『退出』按钮
- 任务是 **detach 后台跑**，即使把 GUI 完全退出，worker 也会继续跑（跟 `pipeline.exe submit` 一样）

## 呼出状态浮层

- **托盘右键 → 显示状态**
- 或按下**全局快捷键**（默认 `Ctrl+Alt+P`）

浮层内容：

```

========================================================
  [ OK  ] pipeline.exe           运行中   PID 12345   内存 44.6 MB
  [ OK  ] extract_frames.exe     运行中   PID 23456   内存 128.3 MB
  [ --  ] dedupe_pic.exe         未运行
--------------------------------------------------------

  状态      : running    worker PID: 12345

  最后消息  : 抽帧子目录 [4/12] sub_004
```

每 5 秒自动刷新，也可点『立即刷新』。

## 查看日志（tail -f）

点主界面『查看日志』按钮，会弹一个 900×560 的日志窗，跟 `pipeline.exe logs -f` 等价：

- 顶部下拉切换 `worker.log`（默认，含子进程 stdout） / `pipeline.log`（只有编排层）
- 首次打开只加载文件末尾 200 KB，防止大日志把 GUI 卡住
- **自动 tail**：默认勾选，每 2 秒读文件末尾增量追加到窗口
- 用户滚到中间时不会强制拉到底；滚到最底时自动跟随最新一行
- 窗口内最多留 500 行，超了自动裁前面
- 底部按钮：
  - `自动 tail`（勾/取消勾）
  - `立即刷新`（强制重载末尾一段）
  - `打开 job 目录`（资源管理器打开 `<out_root>\.pipeline\jobs\<job_id>\`，能直接看 `manifest.json` / `reports/`）
  - `清屏`（只清窗口，不影响文件）
  - `关闭`（隐藏窗口，不销毁；再点『查看日志』复用同一个窗）
- 切换到新任务后（job_id 变了）自动重新加载

## 常见问题

**Q: 打开报『缺少 xxx.exe』？**
把 `extract_frames.exe` / `dedupe_pic.exe` / `license.lic` 复制到 `C:\Windows\System32\`
（需要管理员）或者跟 `pipe_gui.exe` 放同一目录。

**Q: 快捷键注册失败？**
有的堡垒机 RDP 会话会拦截全局钩子。这时用**托盘图标**呼出状态浮层即可。

**Q: 任务能不能停？**
- 主界面『停止当前任务』按钮
- 或托盘右键『停止当前任务』
- 或命令行 `pipeline.exe stop`

**Q: 跟 `pipeline.exe` 是什么关系？**
GUI 内部直接复用 `pipeline` 的模块（同一个 exe 已经打进去了），不需要外部 `pipeline.exe` 存在。
但为了在命令行也能查任务，建议 `pipeline.exe` 也放到 System32。
