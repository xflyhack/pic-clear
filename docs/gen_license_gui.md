# gen_license_gui.exe 使用说明

license.lic 签发工具的**图形版**，双击运行即可。

## ⚠ 安全提示

- 此 exe **内置签发私钥**（CI 打包时把 `secrets/private.pem` 通过 PyInstaller `--add-data` 一起打进去了）
- 具备签发**任意 license** 的能力
- **仅限内部使用，不要外发**。拿到这个 exe 的人就能签任何指纹的 license，等同于私钥泄漏
- 如果你不需要 GUI 版，用命令行版 `gen_license.py` 更安全（每次要求 `--private-key` 显式指定私钥位置）

## 从哪拿到 exe

1. GitHub 仓库 `Actions` 页面找 `Build Gen License GUI EXE` workflow
2. 最新一次成功构建 → `Artifacts` 下载 `gen-license-gui-windows-exe`
3. 解压得到 `gen_license_gui.exe`

## 使用步骤

1. **双击 `gen_license_gui.exe`**
2. 顶部会提示"内置私钥: xxx"（绿色文字），表示 exe 里私钥已就绪
3. **填指纹**：`XXXX-XXXX-XXXX-XXXX`（会自动大写）
4. **发放给**：写用户名或工号（会写进 license，用于追溯）
5. **到期**：
   - 选『永久 (never)』→ 用不失效
   - 或选『到期日』→ 填 `YYYY-MM-DD`
6. **备注**：可选，任意文字
7. **私钥来源**：默认『使用内置私钥』；如果想换密钥对，选『外部文件』然后浏览选一个 `.pem`
8. **保存到**：默认当前目录/`license.lic`，可改
9. 点『**生成 license.lic**』
10. 弹框显示"签发完成"，把生成的 `.lic` 发给用户即可

## 生成的 license.lic 怎么给用户

- 直接把 `.lic` 文件发给用户（微信/邮件）
- 用户放到他的 `dedupe_pic.exe` / `extract_frames.exe` / `pipeline.exe` **同目录**，重新运行即可

## 常见问题

**Q: 顶部提示"未检测到内置私钥"？**
- 说明你在开发模式（直接跑 py）下运行，且项目里没有 `secrets/private.pem`
- 打包后的 exe 里正常会有
- 解决：切换到『外部文件』手动选一个 `.pem`

**Q: 生成后的 license.lic 用不了？**
- 检查两点：
  1. 指纹是否输入正确（跟用户机器的指纹完全一致）
  2. exe 里的**公钥**是不是跟你**签发用的私钥**配对（如果换过密钥对就要重打包所有 exe）
